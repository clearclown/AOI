プロジェクト企画書: Project "Nexus Secretary"

〜AOIプロトコルによる自律型秘書AIメッシュの構築〜

1. 背景と課題

現在、開発現場ではCursorやClaudeCodeなどのAIツールが普及しているが、エンジニアやPMは「自分のAIが生成した情報を相手に伝える」という**「人間ミドルウェア」**の状態に陥っている。

情報の低帯域化: 人間が介在することで、AIが持つ高密度のコンテキスト（設計意図やエラーの詳細）が削ぎ落とされる。

集中力の分断: PMの進捗確認がエンジニアの作業（Deep Work）を中断させる。

認知の限界: AIの進化速度に対し、人間がコードや仕様を理解するスピードが追いつかなくなっている。

2. ビジョン

「人間を介さないAI間の直接対話と操作により、人間を『調整』から解放し、『意志決定』に集中させる」

人間一人ひとりに専属の「秘書AI」を配備し、それらが共通プロトコル（AOI）を用いてネットワーク上で直接交渉・作業代行を行う世界を実現する。

3. 解決策

AOI (Agent Operational Interconnect) プロトコルの提唱

AIエージェントが他のエージェントを識別し、権限の範囲内で情報のクエリや操作のトリガーを引くためのセキュアな通信規格を策定する。

秘書AI（Secretary Agent）の導入

作業用AI（Doer）と外部ネットワークを繋ぐ「外交官」としての秘書層を構築。

PM秘書AI: 進捗管理、仕様の構造化、優先順位の交渉。

エンジニア秘書AI: 作業コンテキストの要約提供、割り込みの遮断、作業AIの操作代行。

4. 期待される効果

会議の削減: 非同期的なAI間通信により、定例会や進捗確認の時間が90%削減される。

生産性の向上: エンジニアは「AIへの入力」や「PMへの説明」から解放され、純粋な設計とレビューに集中できる。

ナレッジの自動同期: ドキュメント化を待たずとも、秘書AI同士の通信により最新の仕様が常にチーム全体で共有される。

5. ロードマップ

Phase 1 (MVP): Tailscale上の閉域網で、エディタのチャットログを要約して共有する「進捗実況」機能の実装。

Phase 2 (Operation): PMのAIからエンジニアの作業AIに対し、テスト実行やドキュメント生成を命じる「操作」機能の追加。

Phase 3 (Ecosystem): 複数社・複数プロジェクトを跨ぐ、エージェント間の「外交・交渉」プロトコルの標準化。
